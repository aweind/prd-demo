# 原型设计准则与交互规范 (README)

> 本文档汇总了与用户沟通确定的原型设计核心要求。旨在作为后续所有新需求原型设计的底层规范，确保交互的一致性与高保定效果。

---

## 1. 布局规范 (Layout)

- **容器尺寸**：统一采用 **1920px** 固定宽度布局（1600px 原型显示区 + 320px PRD 注释面板）。
- **顶层导航**：
    - **背景色**：使用浅灰色（如 `#d6d6d6`）顶栏，以视觉上区分原型内容区。
    - **要素必备**：
        - 左侧功能：**“需求说明”** 触发器（点击展开详细的需求背景与变更点面板）。
        - 中间功能：**“本需求影响页面”** 切换按钮（采用按钮样式的 Tab 切换）。
- **侧边 PRD 面板**：固定位于最右侧，记录交互细则、业务规则及功能点。

---

## 2. 交互体验规范 (UX Patterns)

- **大数据集管理**：
    - 当页面模板或项超过 20 条时，禁止使用多级下拉菜单。
    - **升级方案**：采用 **“管理弹窗 (Modal)”**。弹窗应支持模糊搜索、勾选、批量删除及分页。
- **批量操作**：
    - 在管理列表上方应实现 **“批量操作栏”**（默认隐藏，勾选 Checkbox 后滑出/显现）。
    - 操作栏应清晰展示“已选择 X 项”，并提供删除等批处理入口。
- **高危操作保护**：
    - 删除动作必须触发 **“二次确认弹窗 (Confirm Modal)”**。
    - 确认文案应具备动态性，明确告知删除后受影响的数据（如：关联商品数）。
- **反馈机制**：
    - 弱交互（如复制代码、保存成功）优先使用 **Toast 浮层提示**（参考黑色透明浮层）。
    - 强交互或高风险操作使用 Modal。

---

## 3. UI 视觉规范 (Visual Style)

- **组件风格**：遵循现代企业级后台设计（扁平化、适当的留白、圆角半径 4-8px）。
- **色彩体系**：
    - **Primary**: `#3e7dfb` (主色/品牌色)
    - **Danger**: `#ff4d4f` (危险/删除色)
    - **Status Tags**: 系统级标签（灰白底灰字）、异常/告警标签（红字/黄底）。
- **强制使用骨架屏 (Skeleton Screen)**：界面中**非本需求改动**的元素（如：预装侧栏、固定页眉、无关画布内容）必须全部使用骨架屏占位，以此**降低噪音**，使评审视角聚焦于核心变更。

---

## 4. 技术实现建议

- **代码架构**：优先使用 **原生 HTML / Vanilla CSS / Vanilla JS**。
- **透明度**：所有变量、样式逻辑应集中在 `<style>` 标签中，交互逻辑集中在底部 `<script>` 中。
- **测试友好**：确保关键交互按钮具备明确且唯一的 `id` 或 `class`，方便后续回归走查。

---

## 5. 跨店铺同步 (Special Logic)

- **复制代码**：生成加密 JSON 用于跨店迁移方案（V1/Vogue 主题专属逻辑）。
- **跳转逻辑**：新建成功后，原型应模拟“关闭并跳转编辑页”的完整路径流程。
